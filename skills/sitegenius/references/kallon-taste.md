# Kallon's Taste Profile — learned from real accept/reject decisions

**Purpose**: make first drafts land. This file is the accumulated signal from every build Kallon has reacted to. Read it BEFORE picking a direction; update it after EVERY accept/reject/complaint — a taste call that lives only in chat history is lost when the session ends.

**How to update**: append to the ledger, then revise the profile summary if the new data changes it. Never delete ledger rows — contradictions are signal too (taste evolves, and context matters).

## Profile summary (current best model)

**Loves — lead with these:**
- **Scroll-driven centerpiece moments.** The two strongest positive reactions in project history: v2's pinned panel-stack ("that's good") and the horizontal scroll-jack product rail (the ONE thing he liked from a build he otherwise rejected — "when you scroll down it scrolls the slider"). If a build has no scroll-driven set piece, he reads it as "no animations at all," even when reveals/hover/cursor motion is everywhere.
- **Big confident display type.** Every accepted build has a large serif or expanded-sans headline moment.
- **Soft, gradual transitions.** Hard color cuts between sections read as "pretty harsh" to him; the scroll-linked background crossfade was accepted immediately ("perfect").
- **Video backgrounds.** Asked for them proactively, twice (immigration v3-video, MERIDIAN). Generated video heroes are a reliable win.
- **Seeing range.** He explicitly tests capability ("I really just want to test the capabilities of the skill", "make it very different", "I want to see what your potential is!"). Repeating a technique across consecutive builds disappoints him even when each build is individually fine.

**Rejects — do not resubmit these:**
- **Sidebar navigation layouts** ("don't like the bar on the side") — rejected twice, wants standard top header.
- **Hotspot/pin interactions on imagery** (FIELDWORK) — read as gimmicky.
- **Technical/mono aesthetic** (JetBrains Mono labels, spec-sheet layout, cobalt/tangerine) — "everything looked trash."
- **Under-animated pages** — restraint reads as absence to him. When in doubt, add the centerpiece, don't subtract.
- **Multi-file deliverables** — anything he downloads must be ONE self-contained file that opens and fully works.

**Neutral/kept without comment** (fine but not proven loves): bottom-sheet cart, quick-view panels, bento gallery grids, drag-carousels.

**Communication style**: reacts fast and bluntly; wants to preview in the browser, not from screenshots; interrupts mid-build when he has a new idea — treat interruptions as scope updates, not noise. Asks to be asked before credits are spent (video/image generation), but once he says "you decide," he means it — don't re-ask.
